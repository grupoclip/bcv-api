# Contributing

Thanks for helping improve BCV Today.

BCV Today is an unofficial static JSON API and bilingual website for Banco
Central de Venezuela exchange rates. The public API is versioned under
`/api/v1/`, and generated API files are committed to the repository so GitHub
Pages can serve them directly.

## Development Setup

Install Python dependencies:

```bash
pip install -r requirements.txt
```

Install Jekyll dependencies:

```bash
bundle install
```

If you want to keep gems local to the repository:

```bash
BUNDLE_PATH=vendor/bundle bundle install
```

## Updating Data Locally

Run the scraper and regenerate API JSON files:

```bash
python main.py
```

This writes generated data under:

- `api/v1/rate.json`
- `api/v1/status.json`
- `api/v1/history.json`
- `api/v1/history/YYYY-MM-DD.json`
- `_data/rate.json`

Only commit generated data when the change is intentional. Avoid committing
unrelated rate refreshes in documentation-only or styling-only changes.

## Previewing The Site

Run Jekyll locally:

```bash
bundle exec jekyll serve --baseurl ""
```

Or, if using a repo-local bundle:

```bash
BUNDLE_PATH=vendor/bundle bundle exec jekyll serve --baseurl ""
```

Then open:

```text
http://127.0.0.1:4000
```

## Validation

Before opening a pull request, run the checks that match your change:

```bash
python3 -m py_compile main.py
node --check assets/js/dashboard.js
node --check assets/js/history.js
python3 -m json.tool openapi.json
python3 -m json.tool api/v1/status.json
BUNDLE_PATH=vendor/bundle bundle exec jekyll build --baseurl ""
```

If a check is not relevant or cannot be run locally, mention that in the pull
request.

## API Compatibility

The only public JSON API is `/api/v1/`.

When changing response fields, endpoint paths, date semantics, or generated
history behavior:

- Update `openapi.json`.
- Update Spanish and English API docs.
- Update `README.md`.
- Update `llms.txt`, `llms-full.txt`, and `.well-known` discovery files when
  machine-readable agent context changes.
- Preserve existing fields whenever possible.

Do not add new unversioned public JSON endpoints under `/api/`.

## Content And Localization

The site is bilingual. User-facing text should usually be updated in both
Spanish and English.

Common text sources:

- `_data/i18n.yml`
- `_data/faq.yml`
- `api/index.md`
- `en/api/index.md`
- Spanish pages at the repository root
- English pages under `en/`

## Code Style

- Keep changes focused and avoid unrelated refactors.
- Use clear names and straightforward control flow.
- Prefer existing project patterns over new abstractions.
- Keep generated JSON consistently formatted with two-space indentation.
- Avoid adding runtime dependencies unless there is a clear need.

## Pull Request Guidelines

A useful pull request includes:

- A short summary of what changed.
- Why the change is needed.
- Any data files regenerated by the change.
- Commands run for validation.
- Screenshots for visible UI changes when practical.

## Security

Do not open public issues with exploit details or secrets. Follow the security
reporting process in `.github/SECURITY.md`.
